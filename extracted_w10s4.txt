2.  Why Information Systems are Needed in SCM
SC has: many processes, many entities, many interactions — all happening simultaneously
Without structured systems: operate in haphazard mode — hard to measure performance or maintain reference points
Information systems bring: transparency + efficiency + structured decision support
Key principle: IS should be selected based on organizational need — not just because a competitor uses it
All IS adoption has cost and revenue implications — requires careful cost-benefit evaluation

3.  Five Needs That Drive ERP Implementation
ERP is not about technology first — it emerges from 5 fundamental operational needs:

1
Planning
Identify tasks for near and distant future across all SC processes
Manufacturing, procurement, materials, logistics, warehousing — all need coordinated plans
Avoids daily decision-making / 'fire fighting' — enables a longer time horizon
2
Execution
Convert plan into daily actions — who does what, with what resources
Inventory sorting, dispatch, invoicing, billing happen every day
Without clear execution, even good plans fail
3
Automation
Repetitive tasks (e.g., reminder emails to vendor) → automate to free up human capacity
Automation reduces non-value-added work
Allows personnel to focus on value-adding activities (design, decisions, client interaction)
4
Monitoring
Track inventory levels, orders dispatched, workforce attendance, equipment status — regularly
Data captured digitally can be monitored via dashboards
Identifies issues before they escalate
5
Control
Act on monitoring data — stop a defective assembly line, adjust a schedule, escalate a delay
Can be manual (alert sent to personnel) or automated (system triggers corrective action)
Closes the loop: plan → execute → monitor → control → replan

★  Fire-fighting (daily reactive decision-making) = the enemy of efficiency. ERP enables proactive planning and automated execution to eliminate it.

4.  What is ERP — Enterprise Resource Planning?
Definition
ERP = Enterprise Resource Planning — a software system (or collection of applications) that manages business processes across an organization
Enterprise = connection of several entities / processes delivering value to a customer — not just one department
Can be a single module or a collection of interconnected modules
Based on: large amounts of transactional data that are interconnected via formulas and rules

How It Can Be Implemented
Procure off-the-shelf: SAP, Oracle, Tally (accounting) — market solutions for specific modules
Build in-house: using open source tools, spreadsheets, programming — viable for specific processes
Cloud-based ERP: increasingly popular — avoids local server dependency, accessible anywhere
Key message: do not procure just because it exists or competitors use it — needs must drive the decision

ERP stems from MRP — Material Requirements Planning — but extends far beyond it:

MRP Key Inputs (for reference)
Bill of Materials (BOM): hierarchical breakdown of what components are needed to build the final product (e.g., automobile → seats + frame + engine → sub-components...)
Inventory records: what stock is currently available
Lead times: how long each process / procurement takes
Demand plan: what needs to be produced and when
MRP output: production/procurement schedule — what to make or buy, in what quantity, by when

7.  ERP Use Cases — How Modules Connect
ERP's power comes from linking modules — data flows automatically between functions:

Context
How ERP Integrates Multiple Functions
Consultancy Firm
Problem: how many consultants needed per quarter given current projects?
ERP connects: project pipeline → consultant skill sets → capacity planning → hiring needs
Also linked to: consultant hours logged → client invoicing → salaries + overtime payroll for support staff
Value: frees consultants from admin — they focus on client delivery; backend is automated
Procurement / Vendor Management
Vendor onboarded onto ERP → receives delivery schedule for next 3 weeks directly via system
Vendor logs dispatch data on their ERP module → company receives confirmation
On arrival: GRN recorded in ERP → triggers accounts payable → invoice generated at vendor side
All transactions stored → enables spend analysis: total spend on this vendor, by category, over time

★  ERP is a system of connected formulas — when one input changes, all downstream outputs update automatically. This is what replaces manual data re-entry across departments.

8.  ERP in Supply Chain Management — Benefits & Challenges
✓  Benefits
✗  Challenges in SC Context
Connects multiple entities on a single digital platform
Onboarding all SC partners is hard — different sizes, different ERP systems, different digital maturity levels
Reduces need for frequent in-person interactions
Small vendors may have no ERP at all — cannot afford or lack technical capability
Creates transparent, accountable business transactions
Process-software fit problem: assembly operations ≠ warehousing operations — same ERP may not handle both
Enables long-range planning rather than daily fire-fighting
Vendor with multiple clients: onboarding onto 3+ different clients' ERP systems is very costly
Automates repetitive tasks — frees personnel for value-adding work
Result: ERP implementation in SC is rarely fully end-to-end — gaps remain between partners
Supports spend analysis, audit trails, compliance documentation
Workaround: use ERP for only specific process subsets; maintain different modules for different partners

Workarounds for SC ERP Gaps
Vendor management investment: help small vendors adopt digital tools, even basic ones
Module-specific adoption: implement ERP only for subset of processes where it integrates well
Delinked modules: use one ERP for specific partners; separate module for others — link where possible
Outcome: ERP in SC is rarely fully end-to-end — but companies progressively close the gaps
